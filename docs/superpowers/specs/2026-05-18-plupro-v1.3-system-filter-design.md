# PluPro v1.3 — HIC/EVS 顶层系统筛选与分组显示

> v1.3 在 v1.1 项目分析触发链路基础上,把已埋好但未消费的 frontmatter `system: HIC | EVS`
> 字段在视图层用起来。控制塔列表加顶层 segmented control(全部 / HIC / EVS / 未分类),
> 「全部」视图下按 system 分组渲染。
>
> 数据模型基本不变(types.ts 加一个枚举),纯视图层增强。

---

## 1. 范围

### 1.1 v1.3 in

- 项目 manifest frontmatter 已有的 `system: HIC | EVS` 字段被 scanner 解析并暴露给视图
- 控制塔列表顶部加 segmented control 4 选 1:全部 / HIC / EVS / 未分类
- 「全部」视图下按 system 分组渲染,组与组之间用 divider 标题区分
- 「HIC / EVS / 未分类」单选视图下只显示对应组项目,组标题隐藏
- 项目排序:同组内按 `manifest.title` 字典序;组之间固定 HIC → EVS → 未分类

### 1.2 v1.3 out(明确推迟)

- 设置面板默认筛选状态保存(留 v1.4)
- system 维度的 AggregateProgress 跨项目汇总(留 v1.4)
- 三种以上 system 类别(目前只 HIC / EVS;未来若有"跨厂商""第三方"等再扩)
- Bases 看板的 system 维度集成(用户可自建,不需插件做)

## 2. 数据模型

### 2.1 新增类型

`src/types.ts` 新增:

```typescript
export type ProjectSystem = 'HIC' | 'EVS';

// '__unassigned__' 是视图层内部状态,不进数据模型
export type SystemFilter = 'all' | 'HIC' | 'EVS' | 'unassigned';
```

### 2.2 ProjectManifest 扩展

```typescript
export interface ProjectManifest {
  // ... 原字段 ...
  system?: ProjectSystem;   // 来自 frontmatter 同名字段
}
```

注:**`system` 不强制必填**。manifest 没填或填非法值时为 `undefined`,在视图层归为"未分类系统"组。

### 2.3 ProjectIndexSnapshot 不扩展

不在 `ProjectIndexSnapshot` 内做 `bySystem: Map<ProjectSystem, ProjectEntry[]>` 之类的索引化。
理由:
- 视图层直接 `for...of snapshot.projects` 迭代 + filter 一次即可,O(n) 完全够用(项目数 < 20)
- 避免索引重复维护

## 3. 架构

### 3.1 ProjectManifestScanner 解析

`src/core/ProjectManifestScanner.ts` 的 `parseManifestFromText`:

```typescript
const VALID_SYSTEMS: ProjectSystem[] = ['HIC', 'EVS'];

// 在 return 内追加:
system: (() => {
  const raw = asString(fm.system);
  return raw && VALID_SYSTEMS.includes(raw as ProjectSystem)
    ? (raw as ProjectSystem)
    : undefined;
})(),
```

非法值(如 `system: legacy`)会被静默回退到 `undefined` — 与 status 字段的 fallback 策略一致。

### 3.2 ControlTowerView 状态管理

新增私有字段:

```typescript
private systemFilter: SystemFilter = 'all';   // 默认显示全部
```

切换 filter 不重扫数据,只重渲染。

### 3.3 ControlTowerView 列表头 segmented control

`renderList` 内,在 `<h3>项目</h3>` + ⟳ 刷新按钮 之后插入 segmented control:

```html
<div class="plupro-filter-bar">
  <button class="plupro-filter-chip is-active" data-filter="all">全部 (N)</button>
  <button class="plupro-filter-chip" data-filter="HIC">🟢 HIC (N)</button>
  <button class="plupro-filter-chip" data-filter="EVS">🟠 EVS (N)</button>
  <button class="plupro-filter-chip" data-filter="unassigned">⚪ 未分类 (N)</button>
</div>
```

各 chip 计数从 `snapshot.projects` 实时聚合(包含 unassigned 系统下的 projects):

```typescript
function countBySystem(snapshot: ProjectIndexSnapshot): Record<SystemFilter, number> {
  const counts = { all: 0, HIC: 0, EVS: 0, unassigned: 0 };
  for (const entry of snapshot.projects.values()) {
    counts.all += 1;
    const key = entry.manifest.system ?? 'unassigned';
    counts[key] += 1;
  }
  return counts;
}
```

### 3.4 ControlTowerView 分组渲染

```typescript
function groupProjectsBySystem(snapshot, filter): Array<{system, entries}> {
  // 按 system 分桶
  const buckets = new Map<SystemFilter, ProjectEntry[]>();
  for (const entry of snapshot.projects.values()) {
    const key = (entry.manifest.system ?? 'unassigned') as SystemFilter;
    if (filter !== 'all' && filter !== key) continue;
    const arr = buckets.get(key) ?? [];
    arr.push(entry);
    buckets.set(key, arr);
  }
  // 桶内排序
  for (const arr of buckets.values()) {
    arr.sort((a, b) => a.manifest.title.localeCompare(b.manifest.title, 'zh-CN'));
  }
  // 桶之间固定顺序:HIC → EVS → unassigned
  const order: SystemFilter[] = ['HIC', 'EVS', 'unassigned'];
  return order
    .map((system) => ({ system, entries: buckets.get(system) ?? [] }))
    .filter((g) => g.entries.length > 0);
}
```

渲染规则:
- `filter === 'all'`:渲染所有非空组,每组前加 divider `<h4>━━ 🟢 HIC 系统(N)━━</h4>`
- `filter !== 'all'`:只渲染对应组,**不**显示 divider(因为只有一组,没必要标题)

### 3.5 「未分类(N)」桶位置不变

底部固定的「📦 未分类(N)」change 桶**不受** system filter 影响 — 它表示"无 project 字段的 change"
是另一个维度,与 system 正交。

但当 `systemFilter === 'unassigned'` 时(项目无 system 字段),底部「未分类 change」桶**仍显示**
所有未分类 change(不按 system 过滤,因为 change 的 system 取自其挂载 project,挂的 project 也可能
没 system 字段)。

## 4. UI 规范

### 4.1 Segmented control 视觉

- 4 个 chip 横排,可换行
- 默认态:浅灰背景
- 激活态:Obsidian `--interactive-accent` 蓝色填充
- hover:`--background-modifier-hover`
- 计数 `(N)` 跟在 chip 文字后

CSS class:
- `.plupro-filter-bar`:容器
- `.plupro-filter-chip`:基础样式
- `.plupro-filter-chip.is-active`:激活态
- 系统专属配色:HIC 用 `--text-success`(绿),EVS 用 `--text-accent`(橙),unassigned 用 `--text-muted`

### 4.2 分组 divider

`<h4>` 元素,文字格式:`━━━ <icon> <system label>(N)━━━`
- HIC: `━━━ 🟢 HIC 系统(N)━━━`
- EVS: `━━━ 🟠 EVS 系统(N)━━━`
- unassigned: `━━━ ⚪ 未分类(N)━━━`

CSS class `.plupro-system-divider`,字号略小,颜色 `--text-muted`,前后留白。

### 4.3 项目排序细节

- 同组内按 `title` 中文字典序(`Intl.Collator('zh-CN')`)
- 组间顺序固定:HIC → EVS → unassigned(从主到次)
- 不允许用户自定义(避免与项目 manifest 数据不一致)

## 5. 核心流程

### 5.1 用户首次打开控制塔

1. `onOpen()` 调 `renderAll(snapshot)`
2. `renderList` 渲染 segmented control(默认 `all` 激活)
3. 调用 `groupProjectsBySystem(snapshot, 'all')`
4. 渲染所有非空组,每组前显示 divider
5. 项目卡片点击逻辑、徽章逻辑保持 v1.1 不变

### 5.2 用户点击 HIC chip

1. chip click handler 改 `this.systemFilter = 'HIC'`
2. 调 `this.renderAll(this.plugin.getIndex())` 触发重渲染
3. `renderList` 重新跑 `groupProjectsBySystem(snapshot, 'HIC')`
4. 只渲染 HIC 组,不显示 divider(单组)
5. 其他 chip 切换为非激活态,HIC chip 激活

### 5.3 数据刷新时保留 filter 状态

`refreshIndex()` 触发 `renderAll(newSnapshot)` 时,`this.systemFilter` 状态保持不变,
filter 持续生效。

## 6. 错误处理

| 场景 | 行为 |
|------|------|
| `system: legacy` 等非法值 | scanner 静默回退 `undefined`,项目归「未分类」组 |
| 所有项目都没 system 字段 | HIC/EVS chip 显示 (0),unassigned chip 显示 (N);默认 all 视图全部显示在「未分类」divider 下 |
| 切换到空组(如 HIC chip 但项目都是 EVS) | 显示"无项目"占位提示,保留 chip 状态 |
| filter 状态在视图关闭后重新打开 | 重置回 `all`(v1.3 不持久化,留 v1.4) |

## 9. 已确认的关键决策

| 决策点 | 选择 | 理由 |
|--------|------|------|
| 数据模型扩展 | types.ts 仅加 2 个 type alias,不动 ProjectIndexSnapshot | O(n) 视图层 filter 完全够用,避免索引重复维护 |
| 分组渲染策略 | 「全部」视图分组 + 单系统视图不分组 | 单组没必要标题 |
| 组间顺序 | 固定 HIC → EVS → unassigned | 避免用户自定义带来的不确定性 |
| 桶内排序 | 中文字典序 | 中文项目名最自然 |
| 非法 system 值 | 静默回退 undefined | 与 status 字段一致的容错策略 |
| 持久化 filter 状态 | v1.3 不做,默认 all | 简单,留给 v1.4 + 设置项 |
| 跨厂商等第三类 system | 不支持 | 暂时 HIC/EVS 二分够用 |
